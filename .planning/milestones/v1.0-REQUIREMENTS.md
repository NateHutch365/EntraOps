# Requirements Archive: EntraOps GUI v1.0

**Archived:** 2026-03-26
**Milestone:** v1.0 EntraOps GUI
**Core Value Delivered:** A user who has run `Save-EntraOpsPrivilegedEAMJson` can open a browser and immediately understand who holds ControlPlane access in their tenant — without writing a KQL query, opening Azure Portal, or reading raw JSON.

---

## v1 Requirements — All Complete

### Foundation & Scaffold

- [x] **FOUND-01**: Project scaffold exists at `gui/` with `client/`, `server/`, `shared/` layout and all dependencies installable via `npm install`
- [x] **FOUND-02**: Single command `npm run dev` starts both Vite dev server and Express backend concurrently
- [x] **FOUND-03**: Design system is configured — Tailwind v4, shadcn/ui components, Microsoft Fluent colour tokens, Segoe UI Variable font
- [x] **FOUND-04**: Express server binds to `127.0.0.1` only, with security baseline (helmet, path traversal guards on all file-serving routes)
- [x] **FOUND-05**: React Router v7 app shell with persistent sidebar navigation linking to all top-level pages
- [x] **FOUND-06**: Vite proxies API calls to Express backend in development; build output serves from Express in production
- [x] **FOUND-07**: Shared TypeScript types in `gui/shared/types/` used by both client and server (no type drift)
- [x] **FOUND-08**: Cross-platform compatibility verified on Windows, macOS, and Linux (path separators, PowerShell executable name)

### Dashboard

- [x] **DASH-01**: Dashboard displays privileged object counts broken down by EAM tier (ControlPlane, ManagementPlane, UserAccess)
- [x] **DASH-02**: Dashboard shows object type breakdown per tier (User, Group, Service Principal)
- [x] **DASH-03**: Dashboard shows RBAC system breakdown per tier (EntraID, ResourceApps, IdentityGovernance, DeviceManagement, Defender)
- [x] **DASH-04**: Dashboard shows PIM assignment type chart — Permanent vs Eligible counts per tier
- [x] **DASH-05**: Dashboard shows a "recent changes" widget listing the last 5 git commits that touched `PrivilegedEAM/`
- [x] **DASH-06**: Dashboard displays data freshness — timestamp of the most recently written `PrivilegedEAM/` file
- [x] **DASH-07**: Dashboard shows an empty state with setup instructions when no `PrivilegedEAM/` data exists yet

### Object Browser

- [x] **OBJ-01**: Object browser displays all privileged objects in a sortable, paginated table
- [x] **OBJ-02**: User can filter objects by tier (multi-select), RBAC system (multi-select), object type (multi-select), PIM assignment type, on-premises sync status
- [x] **OBJ-03**: User can search objects by free-text (display name / UPN)
- [x] **OBJ-04**: Filter and search state is reflected in the URL so views are bookmarkable and shareable
- [x] **OBJ-05**: Clicking an object opens a detail panel showing: identity card, all role assignments with classification details, Administrative Unit memberships, Restricted Management AU status, owners, owned objects
- [x] **OBJ-06**: Role assignments in the detail panel are expandable to show the full list of `RoleDefinitionActions` (permissions)
- [x] **OBJ-07**: Each object has a full-page detail view accessible via a stable URL (navigable directly)

### Classification Template Editor

- [x] **TMPL-01**: Template editor presents all five classification files in a tabbed interface: AadResources, AppRoles, Defender, DeviceManagement, IdentityGovernance
- [x] **TMPL-02**: Each template is displayed as a structured tree grouped by tier (ControlPlane, ManagementPlane, UserAccess) then by Category and Service
- [x] **TMPL-03**: User can edit the `RoleDefinitionActions` list for any definition entry inline
- [x] **TMPL-04**: Changes are validated against the expected JSON schema before saving (Zod runtime validation)
- [x] **TMPL-05**: Saving writes back to the corresponding `Classification/Templates/*.json` file on disk
- [x] **TMPL-06**: `Classification/Global.json` exclusion list is viewable and editable in the same UI
- [x] **TMPL-07**: The UI warns the user after saving that edits should be committed to git

## v2 Requirements Delivered Early in v1.0

The following requirements were originally deferred to v2 but were delivered in this milestone:

### PowerShell Command Runner (originally v2 — delivered in Phase 3)

- [x] **RUN-01**: Command palette listing available EntraOps cmdlets (13 allowlisted)
- [x] **RUN-02**: Each cmdlet exposes a parameter form (RBAC system multi-select, boolean toggles, text fields)
- [x] **RUN-03**: Running a command streams output to a terminal-style display in real time with ANSI colour support
- [x] **RUN-04**: Only allowlisted cmdlets can be triggered (security boundary — `z.enum(ALLOWLISTED_CMDLETS)` first gate)
- [x] **RUN-05**: Only one command can run at a time; a "Stop" button kills the running process
- [x] **RUN-06**: Recent command history (session) shown below the terminal; persisted to `gui/.entraops-run-history.json`

### Connect & Classify Setup (originally v2 — delivered in Phase 4)

- [x] **CONN-01**: Sidebar-accessible Connect page with tenant form, auth type selector, and RBAC checklist
- [x] **CONN-02**: Connect streams `Connect-EntraOps` output (device code URL + code) in real time
- [x] **CONN-03**: After auth, classification runs via `Save-EntraOpsPrivilegedEAMJson` with selected RBAC systems, streaming to same terminal view
- [x] **CONN-04**: On completion, toast fires with "Go to Dashboard" CTA; dashboard loads fresh data on navigation

### Git Change History (originally v2 — delivered in Phase 5)

- [x] **HIST-01**: List git commits touching `PrivilegedEAM/` with date, message, and files changed
- [x] **HIST-02**: Selecting a commit shows structured change summary for selected RBAC system's aggregate JSON
- [x] **HIST-03**: Structured change summary: objects added, objects removed, tier changes
- [x] **HIST-04**: User can compare any two commits (not just adjacent ones) with raw diff viewer
- [ ] **HIST-05**: Empty state when `PrivilegedEAM/` has no git history yet — **deferred to v2** (not implemented)

### Settings (originally v2 — delivered in Phase 6)

- [x] **SETT-01**: Display `EntraOpsConfig.json` in a structured (non-raw) form with 5 section cards
- [x] **SETT-02**: Allow editing and saving of `EntraOpsConfig.json` with Zod validation and atomic writes
- [x] **SETT-03**: Empty state when no config file exists yet, with live `New-EntraOpsConfigFile` PowerShell command generator
