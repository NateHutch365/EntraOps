# Milestone v1.2: Self-Service Implementation Workflow

**Status:** ✅ SHIPPED 2026-04-04
**Phases:** 9–12
**Total Plans:** 10

## Overview

Turned the GUI into a point-and-click implementation tool. Admins can now connect, review recommended classification, exclude specific objects, and apply the full tier structure to Entra — all without touching a terminal or a JSON file. Includes real-time SSE streaming progress, pre-run confirmation, and a dry-run / -SampleMode simulation toggle.

### Phase 11: Implementation Workflow

**Goal**: Admins can apply the full EntraOps tier structure to their Entra tenant from the browser with full visibility of what will run
**Depends on**: Phase 10 (Exclusions complete; all pre-apply review screens done)
**Requirements**: IMPL-01, IMPL-02, IMPL-03, IMPL-04, IMPL-06, IMPL-07
**Plans**: 2 plans

Plans:
- [x] 11-01: ApplyPage with 4-state workflow (action selection, confirmation, SSE execution, outcome) + routing + nav
- [x] 11-02: Cross-page CTAs on ObjectBrowser and ReclassifyPage + human-verify

**Details:**

ApplyPage (`/apply`): 4-state machine — Screen 1: action checkboxes (4 EntraOps cmdlets: AU, CA Groups, Unprotected AUs, ControlPlane Scope), Screen 2: confirmation table (#, Action, Cmdlet, Parameters), Screen 3: SSE streaming log (lines appended, Stop button, status indicator), Screen 4: outcome table with per-cmdlet pass/fail. CTA buttons added to Object Browser and Reclassify pages. Human UAT: 8/9 automated confirmable + 1 E2E requiring live Entra session (HUMAN-UAT.md marked passed after browser verification).

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**
- No confirmation dialog on Exclusions Remove (D-01/D-02) — single click calls DELETE immediately; optimistic row removal
- Info banner only after first removal (`hasRemovedOne` state) — not on initial load
- Synchronous pending-map removal before POST on ReclassifyPage Exclude (D-14) — Save count updates immediately before network round-trip
- 4-state machine for ApplyPage (select → confirm → run → outcome) — clean linear flow, no modal nesting
- Pre-run confirmation shows Cmdlet + Parameters only (not explicit Tier Scope column) — tier scope inferable from cmdlet names
- Server-side history guard for dry-run (`if (!isDryRun)`) — not client-side — prevents accidental history pollution
- GlobalExclusionsTab simplified to read-only with link (vs. full mutation UI) — single source of truth for exclusion management
