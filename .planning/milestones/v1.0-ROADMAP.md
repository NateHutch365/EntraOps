## Overview

EntraOps GUI starts as a local-only React dashboard that lets security administrators explore classified EAM data from `PrivilegedEAM/` JSON files without touching Azure Portal or writing KQL. The MVP (Phases 1–2) delivers a fully working read-only dashboard, paginated object browser, and a safe classification template editor — enough to validate the core value proposition. Post-MVP phases (3–6) layer in PowerShell command execution, an in-browser connect & classify wizard, git change history browsing, and a settings editor on top of the proven foundation.

## Key Accomplishments

1. **Working local dashboard from `npm run dev`** — security admins can explore ControlPlane / ManagementPlane / UserAccess tier counts, RBAC breakdown, PIM charts, and data freshness in a browser with zero Azure dependency
2. **Safe classification template editor** — in-browser editing of all 5 EAM tier templates and Global.json with Zod schema validation, diff preview, and atomic writes — eliminating manual JSON hand-editing risk
3. **PowerShell command runner with security allowlist** — 13 explicitly-allowlisted EntraOps cmdlets, real-time SSE streaming with ANSI colour, one-at-a-time concurrency gate, and session history persistence
4. **Connect & Classify wizard** — full device code authentication flow plus classification run entirely from the browser, with sidebar connection status indicator
5. **Git change history browser** — paginated commit list, structured object-level change summaries (added/removed/tier-changed), and any-two-commit comparison with raw diff viewer
6. **Structured settings editor** — `EntraOpsConfig.json` editing via 5-section form with Zod validation, diff preview, CronPicker, safe empty state, and atomic writes

## Milestone Summary

### Key Decisions

- **Decision: Local-only, file-based architecture** — GUI reads from `PrivilegedEAM/` files produced by PowerShell; does not call Microsoft Graph directly. Rationale: eliminates cloud dependency, works offline, respects existing EntraOps security model.
- **Decision: React + TypeScript + Vite + Express + Tailwind v4 + shadcn/ui** — Established pre-milestone in PRD; no deviation. Rationale: modern, well-supported, developer-familiar stack with good TypeScript support.
- **Decision: Security allowlist for PowerShell runner** — Only 13 explicitly-defined cmdlets can be triggered; `z.enum(ALLOWLISTED_CMDLETS)` is the first Zod gate before any process spawn. Rationale: prevents arbitrary code execution from the browser.
- **Decision: DNS-rebinding and path traversal guards on all file routes** — `helmet`, host header validation, and `assertSafePath` on every file-serving endpoint. Rationale: OWASP Top 10 compliance for a tool that reads and writes files from the browser.
- **Decision: Dashboard refresh is manual-via-toast after classify** — No silent auto-refresh after classification completes. Rationale: avoids stale-data race conditions; user explicitly navigates when ready.
- **Decision: Atomic writes for all file mutations** — `.tmp → fs.rename` pattern throughout (templates, global.json, config.json). Rationale: prevents partial-write corruption if process is interrupted.
- **Decision: REPO_ROOT via `import.meta.dirname`** — File-relative resolution, not CWD-relative. Rationale: broke when invoked from non-server directories; fixed in Phase 1.
- **Decision: Same-milestone delivery of v2 requirements** — RUN, CONN, HIST, SETT requirements originally deferred to v2 were all delivered in v1.0. Rationale: scope creep accepted; all features fit naturally on the Phase 1-2 foundation.

### Issues Resolved During Milestone

- **CWD-relative REPO_ROOT** — Broke file reading when `npm` was invoked from outside `gui/server/`; fixed to `import.meta.dirname`.
- **git log parser Invalid Date** — Raw git log parsing using line-by-line split produced misaligned columns; rewritten to use 40-char SHA hash as a commit boundary sentinel.
- **Pagination stale count when RBAC filter active** — `HistoryPage` hides pagination controls when a filter is applied to avoid stale total-count display.
- **Preview Changes button silent no-op** — `handlePreview` only opened dialog if `dirtyActions[key]` existed; silently did nothing if no edits made. Identified as UX gap (no feedback) — tracked as known issue.
- **Terminal line spacing on RunCommandsPage** — Fixed via `leading-normal` class and `\r` stripping before `ansi-to-html` conversion.

### Issues Deferred

- **ConnectPage classification terminal double-spacing** — `\r` stripping and `\n` collapsing implemented, `leading-normal` applied, but visual gaps persist at runtime on the classify step. Root cause unresolved. Tracked: `.planning/todos/pending/2026-03-26-terminal-line-spacing-double-space-powershell-output.md`
- **HIST-05: Empty state when no git history** — Not implemented; deferred to v2.
- **Preview Changes silent no-op (no disabled state or toast)** — Identified in Phase 2 verification but not fixed. Deferred.
