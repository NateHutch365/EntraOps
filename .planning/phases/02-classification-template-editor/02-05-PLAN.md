Complete Classification Template Editor (Plans 02-01 through 02-04):
- Six-tab interface: 5 classification template files + Global Exclusions
- Tier accordion with Category/Service entry cards
- Inline chip editor for RoleDefinitionActions
- Zod-validated Preview→Diff→Confirm→Save flow with atomic file writes
- Global.json GUID list editor with UUID validation
- Post-save git commit warning banner (dismissible, reappears after next save)

  <how-to-verify>
Open: **http://localhost:5173/templates**

**Step 1 — Navigation (TMPL-01)**
- [ ] "Templates" appears in the left sidebar and is highlighted when you're on this page
- [ ] You see 6 tabs: AadResources, AppRoles, Defender, DeviceManagement, IdentityGovernance, Global Exclusions

**Step 2 — Structured tree display (TMPL-02)**
- [ ] Activate "AadResources" tab — it loads (brief spinner, then content)
- [ ] You see an accordion with ControlPlane, ManagementPlane, UserAccess sections
- [ ] Expand ControlPlane — confirm it shows multiple Category/Service entry cards (not raw JSON)
- [ ] Each card shows a scope badge (e.g. "/*") and a list of action chips

**Step 3 — Edit RoleDefinitionActions (TMPL-03)**
- [ ] Open one entry card — see chipEditor with dismissible action pills and an "Add action" input
- [ ] Click × on any chip — that action disappears from the chip list
- [ ] Type a new action string into the input (e.g. `microsoft.directory/test/action`) and press Enter — it appears as a new chip
- [ ] Type an empty string and press Enter — nothing happens (no blank chip added)

**Step 4 — Zod validation gate (TMPL-04)**
*(Note: structural validation failures are hard to trigger from UI alone — skip if not easily testable)*
- [ ] With a valid edit, click "Preview Changes" — DiffDialog opens showing a green line for the added action and a red line for any removed one
- [ ] Click "Cancel" — dialog closes and your edit is still there

**Step 5 — Save to disk (TMPL-05)**
- [ ] With your edit visible, click "Preview Changes" → "Confirm & Save"
- [ ] Dialog closes; the change is reflected in the chip list (saved state)
- [ ] Confirm the file actually changed on disk:
  ```bash
  cat Classification/Templates/Classification_AadResources.json | grep "microsoft.directory/test/action"
  ```
  (or whichever action you added)

**Step 6 — Git warning banner (TMPL-07)**
- [ ] After saving, a yellow banner appears at the top: "Changes saved to disk — remember to commit to git…"
- [ ] Click × on the banner — it disappears
- [ ] Make another save — the banner reappears

**Step 7 — Global Exclusions (TMPL-06)**
- [ ] Click the "Global Exclusions" tab
- [ ] You see the current exclusion list (empty or populated from Global.json)
- [ ] Type `not-a-uuid` in "Add GUID" → error message shown
- [ ] Type `12345678-1234-1234-1234-123456789012` → it appears as a row
- [ ] Click the trash icon → row removed
- [ ] Add another GUID, click "Preview Changes" → DiffDialog shows diff of Global.json
- [ ] Confirm & Save → GUID persists; SaveBanner appears
  </how-to-verify>
