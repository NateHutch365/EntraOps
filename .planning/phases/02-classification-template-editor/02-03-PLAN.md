**Per-entry state architecture:**
Use a single state map for all dirty action edits (keyed by "tierIndex-entryIndex"):
```typescript
const [dirtyActions, setDirtyActions] = useState<Record<string, string[]>>({});
const [diffOpen, setDiffOpen] = useState<string | null>(null);  // "tierIdx-entryIdx" | null
const [saving, setSaving] = useState(false);
const [validationErrors, setValidationErrors] = useState<Record<string, string>>({});  // key → error message
```

Helper to get current actions for an entry (dirty override or original):
```typescript
function getActions(tierIdx: number, entryIdx: number): string[] {
  const key = `${tierIdx}-${entryIdx}`;
  return dirtyActions[key] ?? tiers[tierIdx].TierLevelDefinition[entryIdx].RoleDefinitionActions;
}
```

**Updated entry card rendering** (replace read-only pill list section):
```tsx
<CardContent>
  {/* Read-only scope badge stays */}
  <ChipEditor
    actions={getActions(tierIdx, entryIdx)}
    onChange={(newActions) => {
      const key = `${tierIdx}-${entryIdx}`;
      setDirtyActions(prev => ({ ...prev, [key]: newActions }));
      setValidationErrors(prev => { const next = {...prev}; delete next[key]; return next; });
    }}
  />
  {validationErrors[`${tierIdx}-${entryIdx}`] && (
    <p className="text-destructive text-xs mt-1">
      {validationErrors[`${tierIdx}-${entryIdx}`]}
    </p>
  )}
  <Button
    size="sm"
    variant="outline"
    className="mt-2"
    onClick={() => handlePreview(tierIdx, entryIdx)}
  >
    Preview Changes
  </Button>
</CardContent>
```

**`handlePreview(tierIdx, entryIdx)`:**
1. Get `currentActions = getActions(tierIdx, entryIdx)`
2. Check if dirty (actions differ from original) — if not dirty, do nothing (or show a toast "No changes to preview")
3. Build the proposed full tiers array with the edited entry's actions replaced (deep copy, replace only `RoleDefinitionActions` at the specific tier+entry index)
4. Run client-side Zod validation: import `z` from `zod`. Validate that `currentActions` is `z.array(z.string()).safeParse(currentActions)`. This is a basic structural check. If it fails (shouldn't happen normally), set `validationErrors[key]` and return.
5. Set `diffOpen = "${tierIdx}-${entryIdx}"`

**Build `before` and `after` strings for DiffDialog:**
- `before = JSON.stringify(tiers, null, 2)` (original prop)
- Construct `proposedTiers` by deep-cloning `tiers` and replacing `proposedTiers[tierIdx].TierLevelDefinition[entryIdx].RoleDefinitionActions = currentActions`
- `after = JSON.stringify(proposedTiers, null, 2)`

**Render DiffDialog** (once per accordion, controlled by `diffOpen` state):
```tsx
{diffOpen !== null && (() => {
  const [tIdx, eIdx] = diffOpen.split('-').map(Number);
  const proposedTiers = buildProposedTiers(tIdx, eIdx);  // helper function
  return (
    <DiffDialog
      open={true}
      title={`Preview: ${templateName}.json`}
      before={JSON.stringify(tiers, null, 2)}
      after={JSON.stringify(proposedTiers, null, 2)}
      loading={saving}
      onCancel={() => setDiffOpen(null)}
      onConfirm={async () => {
        setSaving(true);
        try {
          const res = await fetch(`/api/templates/${templateName}`, {
            method: 'PUT',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({ tiers: proposedTiers }),
          });
          if (!res.ok) {
            const err = await res.json();
            setValidationErrors(prev => ({ ...prev, [diffOpen]: JSON.stringify(err.error) }));
            setDiffOpen(null);
            return;
          }
          // Success: clear dirty state for this entry, update parent
          const key = diffOpen;
          setDirtyActions(prev => { const next = {...prev}; delete next[key]; return next; });
          setDiffOpen(null);
          onSaved(proposedTiers);
        } finally {
          setSaving(false);
        }
      }}
    />
  );
})()}
```
