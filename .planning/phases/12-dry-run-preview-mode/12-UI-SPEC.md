# Phase 12 — UI Design Contract: Dry-run / Preview Mode

> Visual and interaction contract for the dry-run / preview mode toggle.
> This phase is a **modification of ApplyPage.tsx** — no new routes or pages.
> All design system values are inherited from Phase 11. Only additions and overrides are declared here.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Scope

This phase does **not** create a new route. It modifies `ApplyPage.tsx` from Phase 11 across all four state-machine screens:

| Screen | State | What changes |
|--------|-------|--------------|
| 1 — Action Selection | `idle` | Dry-run toggle section added below action cards |
| 2 — Pre-run Confirmation | `confirming` | Alert swapped for sky-colored simulation alert; Parameters column adds `-SampleMode` |
| 3 — Live SSE Log | `running` | Heading, running badge, progress label, and terminal separators all declare simulation |
| 4 — Outcome Summary | `done` | Permanent simulation banner; outcome heading prefixed "Dry-run:"; per-cmdlet badges change to sky |

**Core safety requirement:** The admin MUST NOT be able to mistake a dry-run run for a real run at any point during screens 2, 3, and 4. The design applies a three-layer signal to prevent this.

## Screen Contracts (Deltas from Phase 11)

### Screen 1 — Action Selection (`state: idle`) — MODIFIED

**New element: Dry-run toggle section**

Placement: Between the action cards block and the primary CTA. Separated above by a `<Separator />`.

```
[Separator]

[Dry-run toggle row]
  [Switch]  "Dry-run mode"          ← text-sm font-medium text-foreground
            "Simulate changes without writing to Entra. Cmdlets run with -SampleMode."
            ← text-xs text-muted-foreground mt-0.5
  [when on] [Badge "◈ Simulation active"]  ← sky color, right-aligned

[Separator]

[selection count helper text]
[Primary CTA]   ← label changes when dry-run is on
```

**Toggle row specification:**
- Container: `flex items-start justify-between gap-4 py-4`
- Left section (label + sub-label):
  - Label: `text-sm font-medium text-foreground`
  - Sub-label: `text-xs text-muted-foreground mt-0.5 max-w-sm`
  - Sub-label copy: `"Simulate changes without writing to Entra. Cmdlets run with -SampleMode."`
- Right section: `flex items-center gap-3 shrink-0`
  - When on: `Badge` with `bg-sky-500/10 text-sky-500 border border-sky-500/30 text-xs` and copy `"◈ Simulation active"`
  - `Switch` component (`id="dry-run-toggle"`, `checked={isDryRun}`, `onCheckedChange={setIsDryRun}`)
- `Switch` is right of the badge (rightmost element)
- The `Label` element (`htmlFor="dry-run-toggle"`) wraps the left text section for accessibility

**Primary CTA label — mode-dependent:**
| Dry-run off | Dry-run on |
|-------------|------------|
| "Review & Apply" | "Review & Simulate" |

The button `variant="default"` and disabled-when-0-selected logic are unchanged. Only the label text changes.

**Selection counter — unchanged:**
- "N of 4 actions selected" — same copy regardless of dry-run state

### Screen 3 — Live SSE Log (`state: running`) — MODIFIED

All modifications are dry-run-conditional.

**Status row heading — mode-dependent:**
| Mode | Heading copy |
|------|-------------|
| Live | "Applying to Entra…" |
| Dry-run | "Simulating — Dry-run in progress…" |

**Running status badge — mode-dependent:**
| Mode | Badge copy | Badge classes |
|------|-----------|--------------|
| Live | "● Running" | `bg-amber-500/20 text-amber-400 border-amber-500/30` |
| Dry-run | "◈ Simulating" | `bg-sky-500/10 text-sky-500 border-sky-500/30` |

**Progress sub-label — mode-dependent:**
| Mode | Copy |
|------|------|
| Live | "Running action N of M — {Action Label}" |
| Dry-run | "Simulating action N of M — {Action Label}" |

**Persistent dry-run reminder banner (Screen 3, dry-run only):**

Placed immediately above the `TerminalOutput` component when `isDryRun` is true:

```
[Alert sky — compact]
  "Dry-run mode active — no changes are being written to Entra."
```

- `Alert` variant, class: `border-sky-500/30 bg-sky-500/5 py-2`
- Icon: `FlaskConical` size 14, `text-sky-500`
- Body copy: `"Dry-run mode active — no changes are being written to Entra."` — `text-xs text-sky-700 dark:text-sky-400`
- No title (compact single-line format)

**Terminal separator format in dry-run:**

Phase 11 uses: `"─── {FriendlyName} · {timestamp} ───"` in `text-zinc-500`

In dry-run, prefix with `[DRY RUN]`:
```
─── [DRY RUN] Administrative Units · 14:32:01 ───
```
The `[DRY RUN]` prefix is styled `text-sky-500 font-semibold` within the separator. The rest of the separator remains `text-zinc-500`.

This is injected by the client into `htmlContent` before appending each cmdlet's output — same injection point as the existing zinc separator.

### Screen 4 — Outcome Summary (`state: done`) — MODIFIED

**Persistent simulation banner (dry-run only):**

Placed between the heading row and the per-cmdlet results list when `isDryRun` is true:

```
[Alert sky]
  Title: "Dry-run complete — no changes were made"
  Body:  "This was a simulation run. Your Entra tenant was not modified.
          To apply real changes, return to the Apply screen and disable dry-run mode."
```

- Component: `Alert` (shadcn)
- Custom class: `border-sky-500/30 bg-sky-500/5 mb-4`
- Icon: `FlaskConical` size 16, `text-sky-500`
- Title: `text-sm font-medium text-sky-700 dark:text-sky-400`
- Body: `text-sm text-sky-700/80 dark:text-sky-400/80`

**Overall verdict heading — dry-run variants:**

| Condition | Icon | Heading | Status badge |
|-----------|------|---------|--------------|
| All pass | `CheckCircle2` (sky-500) | "Dry-run Complete — No changes made" | `◈ Simulated` sky |
| Any fail | `AlertCircle` (amber-500) | "Dry-run Finished — Review Simulation Results" | `◈ Simulated` sky |
| All fail | `XCircle` (red-500) | "Dry-run — Simulation Errors" | `◈ Simulated` sky |
| Stopped | `OctagonX` (zinc-500) | "Dry-run Stopped" | `◈ Simulated` sky |

Note: The "overall result" icon color follows the actual simulation outcome (sky for all-pass, amber for partial, red for all-fail, zinc for stopped) — only the heading text and the status `◈ Simulated` badge are sky. This preserves meaningful outcome color signals while keeping simulation identity visible.

The `◈ Simulated` status badge:
- Classes: `bg-sky-500/10 text-sky-500 border border-sky-500/30 text-xs`
- Copy: `"◈ Simulated"`
- Replaces the green `"✓ Complete"` / amber / red / zinc outcome badges that appear in live mode

**Per-cmdlet results list — dry-run badge variants:**

| Condition | Badge copy | Badge classes |
|-----------|-----------|--------------|
| Pass (simulated) | "◈ Sim. Pass" | `bg-sky-500/10 text-sky-500 border-sky-500/30` |
| Fail (simulated) | "◈ Sim. Fail" | `bg-red-500/20 text-red-400 border-red-500/30` |
| Skipped | "◼ Skipped" | `bg-zinc-500/20 text-zinc-400 border-zinc-500/30` (unchanged) |

Rationale: "Sim. Pass" uses sky to reinforce simulation — not the live green. "Sim. Fail" retains red because a cmdlet failure during simulation is still a real failure indicator. Skipped is unchanged.

**Button row — dry-run mode:**

| Button | Label | Variant | Action |
|--------|-------|---------|--------|
| Reset to apply | "Apply Again" | `outline` | Resets to `idle`, preserves previous action selections AND preserves `isDryRun` state |
| Navigate away | "Back to Dashboard" | `default` | `navigate('/')` |

`isDryRun` is preserved on "Apply Again" so the admin can immediately simulate again without re-enabling the toggle. They must explicitly turn the toggle off to switch to a live run — this is intentional.

## Three-Layer Safety Design

The admin CANNOT mistake a dry-run for a real run because:

| Layer | Where | Signal |
|-------|-------|--------|
| 1. Mode anchor | Screen 1 toggle | "◈ Simulation active" badge visible before committing |
| 2. Branding switch | Screens 2, 3, 4 headings and CTAs | Every heading, CTA, badge, and label uses "Dry-run" / "Simulating" language |
| 3. Persistent sky banner | Screens 2, 3, 4 | Sky `Alert` always visible at page level to remind admin of mode |

No live-mode colors (amber warning, green success) appear in dry-run outcome screens. Sky color is reserved exclusively for simulation identity.
