<context>
MVP is delivered (Phases 1 + 2 complete). Phase 3 is the PowerShell Command Runner — the most security-sensitive phase in the roadmap. Key concerns to keep in mind when planning:

1. **Strict allowlist** — only explicitly defined EntraOps cmdlets can be triggered; no arbitrary code execution (RUN-01)
2. **Real-time streaming** — output streams to a terminal-style UI with ANSI colour, with a Stop button to kill the process (RUN-02)
3. **Single-run gate** — only one command at a time; second attempt blocked with option to stop current (RUN-03)
4. **macOS/Linux pwsh** — the server runs `pwsh` (PowerShell Core); check `which pwsh` availability
5. **Session history** — command history visible below the terminal output

The existing Express server architecture (gui/server/) is the foundation. The streaming will likely use Server-Sent Events (SSE) or WebSockets. SSE is simpler for one-directional output streaming.

CODEBASE CONCERNS.md at `.planning/codebase/CONCERNS.md` has security notes worth reviewing before Phase 3 planning.
</context>
