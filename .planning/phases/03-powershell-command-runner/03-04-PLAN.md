    Run `npm run dev` from gui/ and visit http://localhost:5173

    **Step 1 — Navigation:**
    Visit / (Dashboard) — confirm "Run Commands" appears in the sidebar nav with a Terminal icon.
    Click it — confirm you land at /run.

    **Step 2 — Command palette:**
    The command runner page should show a command palette/selector. Confirm all 13 cmdlets are listed:
    Save-EntraOpsPrivilegedEAMJson, Save-EntraOpsWorkloadIdentityInfo,
    Save-EntraOpsPrivilegedEAMWatchLists, Save-EntraOpsPrivilegedEAMInsightsCustomTable,
    Save-EntraOpsPrivilegedEAMEnrichmentToWatchLists, Save-EntraOpsWorkloadIdentityEnrichmentWatchLists,
    Update-EntraOpsPrivilegedAdministrativeUnit, Update-EntraOpsPrivilegedConditionalAccessGroup,
    Update-EntraOpsPrivilegedUnprotectedAdministrativeUnit, Update-EntraOpsClassificationControlPlaneScope,
    Update-EntraOpsClassificationFiles, Update-EntraOpsRequiredWorkflowParameters, Update-EntraOps

    **Step 3 — Parameter form:**
    Select `Save-EntraOpsPrivilegedEAMJson` — confirm the form shows:
    - 5 RBACSystem checkboxes (EntraID, ResourceApps, IdentityGovernance, DeviceManagement, Defender)
    - SampleMode boolean toggle
    - TenantName and SubscriptionId text fields (pre-populated if EntraOpsConfig.json exists)

    **Step 4 — Update-EntraOps confirmation:**
    Select `Update-EntraOps` — click Run — confirm a confirmation dialog appears warning that it will
    update module files on disk. Click Cancel — confirm no run starts.

    **Step 5 — Live run (optional — requires Connect-EntraOps in pwsh session):**
    If you have a connected PS session, select `Update-EntraOpsClassificationFiles` and run it.
    Confirm output streams incrementally in the terminal panel with ANSI colors preserved.
    Confirm the status badge shows `● Running` (amber) during execution and `✓ Complete` (green) after.

    **Step 6 — Stop button (optional):**
    Start a long-running cmdlet and click Stop.
    Confirm the terminal displays `[Stopped by user at HH:MM:SS]` and the status badge shows `◼ Stopped`.

    **Step 7 — History persistence:**
    After any run completes, reload the page (F5).
    Confirm the history entry is still visible.
    Click the entry — confirm the form repopulates with that cmdlet and its parameters.

    **Step 8 — Concurrency gate:**
    While a command is running, try to start another from a different browser tab (same origin).
    Confirm a toast error appears: "A command is already running."

    **Step 9 — Security sanity:**
    Open DevTools Network tab, find the POST to /api/commands/run.
    Confirm `cmdlet` in the request body matches one of the 13 allowlisted names exactly.
    Try editing the request (via Fetch override) to send `cmdlet: "Get-Process"` — confirm server returns 400.
