  <how-to-verify>
**Step 1 — Start the dev server:**
```bash
cd /Users/nathanhutchinson/Dev/EntraOps/gui && npm run dev
```
Both client (port 5173) and server (port 3001) should start. Confirm no startup errors in the terminal.

**Step 2 — Dashboard loads:**
Open http://localhost:5173

One of two outcomes:
a) You have PrivilegedEAM/ JSON files → KPI cards should show actual tier counts, bar chart visible, commits/freshness shown
b) No data files → EmptyState with "No privileged identity data yet" and "Save-EntraOpsPrivilegedEAMJson" command shown

✅ PASS if either renders without error
❌ FAIL if blank page, crash overlay, or console errors about missing components

**Step 3 — Sidebar:**
- Sidebar should show "Dashboard" and "Browse Objects" nav links
- Click the collapse/expand toggle → sidebar should animate between 220px and 56px
- Text labels should hide when collapsed, icons remain visible

✅ PASS if collapse animation works
❌ FAIL if sidebar is always expanded or always collapsed, or layout breaks

**Step 4 — Navigate to Object Browser:**
Click "Browse Objects" in the sidebar OR visit http://localhost:5173/objects

- Table should appear with 6 columns: Display Name, Object Type, Tier, RBAC System, PIM Type, On-Prem Sync
- Filter bar with search input + 5 filter popovers should be visible
- If no data: table shows "No objects match the current filters" empty row

✅ PASS if table loads (with or without data)
❌ FAIL if blank page or console errors

**Step 5 — Filters and URL sync (requires data):**
If you have object data:
- Click "Tier" popover → select "ControlPlane"
- Confirm URL updated: ?tier=ControlPlane
- Confirm table re-fetched and loading overlay appeared briefly
- Confirm filter chip appears below the filter bar with "Tier: ControlPlane"
- Click × on the chip → chip disappears, URL clears the tier param, table re-fetches

✅ PASS if filters update URL and chips are dismissible
❌ FAIL if URL doesn't update or chips don't remove

**Step 6 — Object detail Sheet (requires data):**
Click any row in the object table.

- Sheet should slide in from the right
- Object table should still be visible/scrollable behind it
- Sheet identity card: DisplayName large (20px), UPN/AppId below it in muted text
- ObjectType, Tier, OnPremSync badges visible
- Role assignments grouped by RoleSystem heading
- Clicking a role row should expand it to show permissions (Collapsible)
- If role has > 5 actions: "Show all N actions" button visible

✅ PASS if Sheet opens with correct content
❌ FAIL if Sheet doesn't open or identity card is missing fields

**Step 7 — Full page detail:**
With the Sheet open, click "Open full page →" in the Sheet footer.

- Sheet should close
- URL should navigate to /objects/:objectId
- Same identity card and role assignments should render on the full page
- Press browser Back → should return to /objects

Hard refresh test: With URL at /objects/some-object-id, press Ctrl+R or Cmd+R.
- Page should reload and still show the object detail (not a 404)

✅ PASS if full page works including hard refresh
❌ FAIL if hard refresh returns 404 or blank page

**Step 8 — Dashboard Check Again button (if EmptyState visible):**
If you saw the EmptyState in Step 2, click "Check Again" button.
- It should re-fetch /api/dashboard (check Network tab)
- Should show EmptyState again (since no data was added)

✅ PASS if button triggers a network request
❌ FAIL if button does nothing
