**handleRun function** (sequential multi-cmdlet execution):
```
stoppedRef.current = false
setPageState('running')
setStatus('running')
setHtmlContent('')
const selectedActions = IMPLEMENTATION_ACTIONS.filter(a => selected.has(a.cmdlet))
const newResults = new Map(selectedActions.map(a => [a.cmdlet, 'pending' as CmdletResult]))
setResults(newResults)

const abort = new AbortController()
abortRef.current = abort

for (let i = 0; i < selectedActions.length; i++) {
  if (stoppedRef.current) {
    // Mark remaining as skipped
    for (let j = i; j < selectedActions.length; j++) {
      newResults.set(selectedActions[j].cmdlet, 'skipped')
    }
    setResults(new Map(newResults))
    break
  }
  
  setRunningIndex(i)
  const action = selectedActions[i]
  
  // Inject separator
  const timestamp = new Date().toLocaleString([], { ... })
  const sep = `\n<span class="text-zinc-500">─── ${action.label} · ${timestamp} ───</span>\n`
  setHtmlContent(prev => prev + sep)
  
  // Build parameters
  const parameters: CmdletParameters = {}
  if (tenantName.trim()) parameters.TenantName = tenantName.trim()
  if (subscriptionId.trim()) parameters.SubscriptionId = subscriptionId.trim()
  
  // Run cmdlet via SSE (exact pattern from RunCommandsPage lines 92-145)
  const exitCode = await runSingleCmdlet(action.cmdlet, parameters, abort.signal)
  
  newResults.set(action.cmdlet, exitCode === 0 ? 'pass' : 'fail')
  setResults(new Map(newResults))
}

setPageState('done')
setStatus(determineOverallStatus(newResults))
```

**runSingleCmdlet helper** (extracted SSE consumption — returns exit code or -1):
Reuse exact `fetch + ReadableStream` pattern from RunCommandsPage. Key differences:
- Returns a Promise<number> (the exit code)
- On abort: returns -1
- On 409 (already running): append error HTML, return -1
- On network error: append error HTML, return -1

**handleStop:** Set `stoppedRef.current = true`, abort the current fetch, call `/api/commands/stop`, set status 'stopped'.
