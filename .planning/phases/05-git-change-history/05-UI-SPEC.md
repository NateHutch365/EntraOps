## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 |
| Label | 12px (text-xs) | 600 (semibold) | 1.4 |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 |
| Mono | 12px (text-xs font-mono) | 400 (normal) | 1.4 |

**Phase-specific usage:**
- Commit hash: Mono (12px, font-mono, muted-foreground) — matches RecentCommits widget
- Commit message: Body (14px) — primary row content
- Commit author + date: Label (12px, muted-foreground)
- Tier section header (ControlPlane / ManagementPlane / UserAccess): Label (12px, semibold, uppercase tracking-wide)
- Page title ("Change History"): Heading (20px, semibold)
- Object ID in change rows: Mono (12px, font-mono)
- Role assignment delta: Label (12px) with color-coded prefix

**Accent reserved for:** Compare CTA button, active tab underline/indicator, selected checkbox fill, pagination active page number

**Tier semantic colors (established — reused from globals.css):**

| Tier | Token | Value | Phase 5 Usage |
|------|-------|-------|---------------|
| ControlPlane | `--tier-control` | oklch(0.52 0.22 261) / blue | Section border-left + stronger bg tint on ControlPlane change summary |
| ManagementPlane | `--tier-management` | oklch(0.62 0.18 31) / orange | Section border-left on ManagementPlane changes |
| UserAccess | `--tier-user` | oklch(0.68 0.14 145) / green | Section border-left on UserAccess changes |

**Diff colors (new tokens — add to globals.css):**

| Role | Light Mode | Usage |
|------|-----------|-------|
| Added row | `bg-green-950/10` (Tailwind) | Added object rows, added role assignments |
| Added text | `text-green-700` | "+" prefix on added role assignments |
| Removed row | `bg-red-950/10` (Tailwind) | Removed object rows, removed role assignments |
| Removed text | `text-red-700` | "−" prefix on removed role assignments |
| Tier changed row | `bg-amber-950/10` (Tailwind) | Tier-changed object rows |
| Tier changed badge | `bg-amber-100 text-amber-800` | Old tier → New tier badge |

**ControlPlane visual distinction:** The ControlPlane section in every change summary uses `border-l-4 border-tier-control bg-tier-control/5` — a left border and faint background tint. ManagementPlane and UserAccess sections use only `border-l-2` with their respective tier color, no background tint. This ensures ControlPlane draws the eye first — matching the security-first signal requirement.

## Interaction Patterns

### Commit Row Expand/Collapse
- Click anywhere on the commit row (except checkbox) → toggles inline expansion
- Expanded section loads lazily (Skeleton while fetching)
- Only one commit expanded at a time OR multiple allowed — Claude's discretion (recommend: multiple, since user may want to glance at adjacent commits)

### Checkbox Selection for Compare
- Checkbox on left edge of each commit row
- 0 checked → no sticky bar
- 1 checked → sticky bar shows "Select one more commit to compare" (muted, no CTA button)
- 2 checked → sticky bar shows "Comparing 2 commits — **Compare Commits**" (accent button enabled)
- 3rd check → deselects the oldest of the two (FIFO)
- Checking/unchecking does NOT expand/collapse the row

### Sticky Compare Bar
- Fixed to bottom of viewport (not page bottom)
- Same width as content area (respects sidebar)
- `bg-card border-t shadow-lg` — visually distinct floating bar
- Appears/disappears with CSS transition (opacity + transform)

### RBAC System Tabs
- Same shadcn `Tabs` component and visual pattern as Template Editor
- Only tabs for systems with changes in this commit/range
- Tab order: EntraID | ResourceApps | IdentityGovernance | DeviceManagement | Defender

### RBAC System Filter (History Page)
- Multi-select chip bar above commit list — same Popover+Command pattern as ObjectFilters
- Client-side filter (no re-fetch)
- Dismissible chips with ×

### Pagination
- 20 commits per page
- Prev/Next buttons + page numbers
- Same pattern as Object Browser server-side pagination
- Disabled state on first/last page boundaries

### Raw JSON Diff Toggle (Compare Page)
- Collapsible section below structured summary per RBAC tab
- Button text toggles: "Show raw JSON diff" ↔ "Hide raw JSON diff"
- Renders in `<pre>` with line-by-line diff coloring (green bg for additions, red bg for removals)
- No Monaco — lightweight `<pre>` rendering with Tailwind classes
