```typescript
export interface CommitListItem extends GitCommit {
  fullHash: string;
  affectedSystems: RbacSystem[];
  hasPrivilegedEAMChanges: boolean;
}

export interface CommitListResponse {
  commits: CommitListItem[];
  total: number;
  page: number;
  pageSize: number;
}

export interface CommitChangeSummary {
  commitHash: string;
  rbacSystem: RbacSystem;
  sections: TierSectionChanges[];
}

export interface TierSectionChanges {
  tier: 'ControlPlane' | 'ManagementPlane' | 'UserAccess';
  added: ObjectChange[];
  removed: ObjectChange[];
  tierChanged: ObjectChange[];
}

export interface ObjectChange {
  objectId: string;
  objectDisplayName: string;
  objectType: string;
  changeType: 'added' | 'removed' | 'tierChanged';
  previousTier?: EamTier;
  currentTier?: EamTier;
  roleAssignmentDelta?: RoleAssignmentDelta[];
}

export interface RoleAssignmentDelta {
  action: 'added' | 'removed';
  roleDefinitionName: string;
  tier: EamTier;
}
```

**1. `hooks/useCommits.ts`** — data fetching hook following the same pattern as `useObjects.ts`:
- Takes `{ page: number; pageSize: number }` params
- Fetches `GET /api/git/commits?page={page}&pageSize={pageSize}`
- Returns `{ data: CommitListResponse | null, isLoading: boolean, error: string | null }`
- Uses `useEffect` with `[page, pageSize]` dependency array
- fetch pattern: same as useObjects (check if it uses SWR or plain fetch+useState, follow the same approach)

**2. `components/history/ObjectChangeRow.tsx`** — single object in a change list:
- Props: `{ change: ObjectChange }`
- Renders: objectDisplayName (body text), objectId (mono, text-xs, muted-foreground), objectType badge
- For `tierChanged`: show `previousTier → currentTier` with `Badge` in `bg-amber-100 text-amber-800`
- Below tier changed row: render `roleAssignmentDelta` list:
  - Added: `text-green-700` prefix "+" with roleDefinitionName and tier badge
  - Removed: `text-red-700` prefix "−" with roleDefinitionName and tier badge
- Row background: added → `bg-green-950/10`, removed → `bg-red-950/10`, tierChanged → `bg-amber-950/10`

**3. `components/history/TierSection.tsx`** — one tier's changes:
- Props: `{ section: TierSectionChanges }`
- Renders tier label as uppercase `text-xs font-semibold tracking-wide` header
- ControlPlane: `border-l-4 border-tier-control bg-tier-control/5` (per UI-SPEC)
- ManagementPlane: `border-l-2 border-tier-management` (no bg tint)
- UserAccess: `border-l-2 border-tier-user` (no bg tint)
- Three subsections: "Added" (if any), "Removed" (if any), "Tier Changed" (if any)
- Each subsection: label, then `ObjectChangeRow` for each change
- If section has no changes at all (all three arrays empty): show "No changes" in muted text
- ControlPlane section always renders even if empty — "No ControlPlane changes" message is meaningful per CONTEXT.md

**4. `components/history/RbacSystemTabs.tsx`** — dynamic RBAC system tabs:
- Props: `{ systems: RbacSystem[], commitHash: string, mode: 'single' | 'compare', fromHash?: string, toHash?: string }`
- Uses shadcn `<Tabs>` — only tabs for systems in `systems` prop
- Tab order: follow standard order (EntraID | ResourceApps | IdentityGovernance | DeviceManagement | Defender), filtered to only those in `systems`
- Default tab: first system in the filtered list
- Content per tab: fetch change summary lazily on tab activation
  - For `mode='single'`: `GET /api/git/commits/{commitHash}/changes?rbac={system}`
  - For `mode='compare'`: receives sections data from parent (compare page fetches differently)
- While loading: show `<Skeleton>` placeholder
- Render `ChangeSummary` with the fetched data

**5. `components/history/ChangeSummary.tsx`** — structured 3-tier change display:
- Props: `{ sections: TierSectionChanges[] }`
- Renders `TierSection` for each of the 3 sections (ControlPlane first, ManagementPlane, UserAccess)
- Separated by `<Separator>`

**6. `components/history/RbacFilterBar.tsx`** — multi-select RBAC filter above commit list:

**7. `components/history/StickyCompareBar.tsx`** — fixed bottom bar:

- Styling: `fixed bottom-0 right-0 left-[var(--sidebar-width)] bg-card border-t shadow-lg p-4 flex items-center justify-between z-50`
