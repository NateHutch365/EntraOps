  <action>
Create `gui/server/services/changeSummary.ts` with two exported pure functions:

**`computeChangeSummary(parentObjects: PrivilegedObject[], currentObjects: PrivilegedObject[]): TierSectionChanges[]`**

Algorithm:
1. Index `parentObjects` into a `Map<string, PrivilegedObject>` keyed by `ObjectId`
2. Index `currentObjects` into a `Map<string, PrivilegedObject>` keyed by `ObjectId`
3. Initialize three `TierSectionChanges` sections: `ControlPlane`, `ManagementPlane`, `UserAccess`
4. Walk `currentObjects` map:
   - If `ObjectId` NOT in parent map → **added**. Classify into the section matching `current.ObjectAdminTierLevelName`. If tier is `Unclassified`, skip (don't add to any section).
   - If `ObjectId` IS in parent map AND `ObjectAdminTierLevelName` differs → **tierChanged**. Classify into the section matching `current.ObjectAdminTierLevelName`. Set `previousTier` and `currentTier`. Compute `roleAssignmentDelta` via `computeRoleAssignmentDelta()`.
5. Walk `parentObjects` map:
   - If `ObjectId` NOT in current map → **removed**. Classify into the section matching `parent.ObjectAdminTierLevelName`. If tier is `Unclassified`, skip.
6. Return the three sections array (always ControlPlane first, ManagementPlane second, UserAccess third — per CONTEXT.md decision B).

**`computeRoleAssignmentDelta(parentObj: PrivilegedObject, currentObj: PrivilegedObject): RoleAssignmentDelta[]`**

Algorithm:
1. Index parent `RoleAssignments` into `Map<string, RoleAssignment>` keyed by `RoleAssignmentId`
2. Index current `RoleAssignments` into `Map<string, RoleAssignment>` keyed by `RoleAssignmentId`
3. For assignments in current but NOT in parent → `{ action: 'added', roleDefinitionName: assignment.RoleDefinitionName, tier: assignment.Classification[0]?.AdminTierLevelName ?? 'Unclassified' }`
4. For assignments in parent but NOT in current → `{ action: 'removed', roleDefinitionName: assignment.RoleDefinitionName, tier: assignment.Classification[0]?.AdminTierLevelName ?? 'Unclassified' }`
5. Return the delta array

**Exported functions:**

**1. `getPaginatedCommits(page: number, pageSize: number): Promise<CommitListResponse>`**

- Uses `git.raw()` to get commits with changed file names in a single call:
  ```
  git log --max-count={pageSize} --skip={(page-1)*pageSize} --format=%H%n%h%n%an%n%aI%n%s --name-only
  ```
  (No pathspec — list ALL repo commits per CONTEXT.md decision A)
- Parse the raw output: each commit block is separated by a blank line. The format lines are: fullHash, shortHash, author, date, subject. Remaining lines until next blank line are changed file names.
- For each commit, derive `affectedSystems` from file names matching `PrivilegedEAM/{System}/`, and set `hasPrivilegedEAMChanges = affectedSystems.length > 0`.
- Get total count via `git.raw(['rev-list', '--count', 'HEAD'])`.
- Return `{ commits: CommitListItem[], total, page, pageSize }`.
- On error (no git, empty repo): return `{ commits: [], total: 0, page, pageSize }`.

**2. `getCommitChangeSummary(hash: string, rbacSystem: RbacSystem): Promise<CommitChangeSummary>`**

- Get current state: `git.show(['{hash}:PrivilegedEAM/{rbacSystem}/{rbacSystem}.json'])` → parse JSON as `PrivilegedObject[]`
- Get parent state: `git.show(['{hash}^:PrivilegedEAM/{rbacSystem}/{rbacSystem}.json'])` → parse JSON as `PrivilegedObject[]`
- If either `git.show()` throws (file doesn't exist at that commit), treat as empty array `[]`
- Call `computeChangeSummary(parentObjects, currentObjects)`
- Return `{ commitHash: hash, rbacSystem, sections }`

**3. `getCommitComparison(fromHash: string, toHash: string, rbacSystem: RbacSystem): Promise<ComparisonResult>`**

- Get "from" state: `git.show(['{fromHash}:PrivilegedEAM/{rbacSystem}/{rbacSystem}.json'])` → parse JSON
- Get "to" state: `git.show(['{toHash}:PrivilegedEAM/{rbacSystem}/{rbacSystem}.json'])` → parse JSON
- Get raw diff: `git.diff([fromHash, toHash, '--', 'PrivilegedEAM/{rbacSystem}/{rbacSystem}.json'])`
- Compute `sections` via `computeChangeSummary(fromObjects, toObjects)`
- Get commit metadata for both hashes via `git.log({ maxCount: 1, from: hash, to: hash })` or `git.raw(['log', '-1', '--format=%h%n%an%n%aI%n%s', hash])` — build `GitCommit & { fullHash }` for each
- Return `{ from, to, rbacSystem, sections, rawDiff }`

**4. `getAffectedSystems(hash: string): Promise<RbacSystem[]>`**

- Uses `git.raw(['diff-tree', '--no-commit-id', '--name-only', '-r', hash])` to get changed files
- Filters to `PrivilegedEAM/` paths, extracts unique RBAC system names
- Returns only systems that exist in the `RBAC_SYSTEMS` constant
- On error: returns `[]`

**Error handling pattern** (same as gitLog.ts):
- Wrap every `git.show()` call in try/catch → return `[]` on error
- Wrap every function body in try/catch → return safe defaults (empty arrays, empty strings)
- Never throw from this service — callers get safe empty defaults
