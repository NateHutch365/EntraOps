## Spacing Scale

Tailwind default 8-point scale. All values are multiples of 4.

| Token | px value | Tailwind class | Usage in Phase 4 |
|-------|----------|----------------|-----------------|
| xs | 4px | `gap-1` / `p-1` | Icon gaps, dot margins |
| sm | 8px | `gap-2` / `p-2` | Inline element spacing, badge padding |
| md | 16px | `gap-4` / `p-4` | Card inner padding, form field gaps |
| lg | 24px | `gap-6` / `p-6` | Wizard card section padding |
| xl | 32px | `gap-8` / `p-8` | Wizard card outer padding |
| 2xl | 48px | `mt-12` | Page top margin for wizard card |
| 3xl | 64px | — | Not used in Phase 4 |

**Exceptions:**
- Sidebar connection status pill: minimum height `44px` (accessibility touch target)
- Step indicator circles: exactly `32px × 32px` (`w-8 h-8`)
- Step connector lines: `h-0.5` (2px) — decorative only
- Sidebar nav active left-border offset: `pl-[6px]` (6px) — inherited carry-over from Phases 1–3 pattern. `border-l-2` occupies 2px; the remaining 4px visual padding equals 1 spacing unit. Visual consistency with existing pages requires this value — changing it would require a cross-phase refactor.

### Component: Step Indicator (custom, no shadcn block)

**Rendered above the wizard card.** Non-interactive — no click-to-navigate.

**Structure:**
```
<div className="flex items-center justify-between mb-8">
  {steps.map((step, i) => (
    <>
      <StepNode index={i} state={getState(i, currentStep)} label={step.label} />
      {i < steps.length - 1 && <ConnectorLine completed={i < currentStep} />}
    </>
  ))}
</div>
```

**Step labels (exact strings):**
1. `"Tenant"`
2. `"Authenticate"`
3. `"Review & Classify"`
4. `"Classifying"`

**StepNode states:**

| State | Circle | Label |
|-------|--------|-------|
| `completed` | `w-8 h-8 rounded-full bg-fluent-accent flex items-center justify-center` — white `Check` icon (16px) | `text-xs text-muted-foreground mt-1` |
| `active` | `w-8 h-8 rounded-full border-2 border-fluent-accent bg-background flex items-center justify-center` — `text-fluent-accent text-sm font-semibold` step number | `text-xs font-semibold text-foreground mt-1` |
| `upcoming` | `w-8 h-8 rounded-full border-2 border-border bg-muted flex items-center justify-center` — `text-muted-foreground text-sm` step number | `text-xs text-muted-foreground mt-1` |

**ConnectorLine:**
```
<div className={cn(
  "flex-1 h-0.5 mx-2",
  completed ? "bg-fluent-accent" : "bg-border"
)} />
```

### Component: Wizard Card

shadcn `<Card>` containing each step. Single card that swaps step content via `currentStep` state.

```
<Card className="shadow-sm">
  <CardHeader className="flex flex-row items-start justify-between pb-4">
    <div>
      <CardTitle className="text-lg font-semibold">{stepHeading}</CardTitle>
      <CardDescription className="text-sm text-muted-foreground mt-1">{stepDescription}</CardDescription>
    </div>
    {/* Cancel button: "Exit Setup" on steps 1–3; "Stop & Start Over" on step 4 running; hidden on step 4 complete/failed */}
    <Button variant="ghost" size="sm" onClick={handleCancel} className="text-muted-foreground hover:text-destructive">
      {cancelLabel}
    </Button>
  </CardHeader>
  <CardContent>
    {/* Step content */}
  </CardContent>
</Card>
```

**Cancel button behaviour by step:**
| Step | Cancel action |
|------|--------------|
| Step 1 (Tenant) | Clear form, stay on step 1 (no-op if nothing entered) |
| Step 2 (Authenticate, streaming) | Abort SSE stream → `POST /api/connect/disconnect` → reset to step 1 |
| Step 2 (Authenticate, completed) | `POST /api/connect/disconnect` → reset to step 1 |
| Step 3 (Review & Classify) | `POST /api/connect/disconnect` → reset to step 1 |
| Step 4 (Classifying, streaming) | Stop classification stream (abort) → reset to step 1 (no disconnect — classify does not need disconnect) |
| Step 4 (Classifying, completed/failed) | Reset to step 1 (wizard complete; cancel = start over) |

### Step 4: Classifying

**Card heading (dynamic):**
- While streaming: `"Classifying your tenant…"`
- On success (`status === 'completed'`): `"Classification complete"`
- On failure (`status === 'failed'`): `"Classification failed"`

**Card description:**
- While running: `"Your tenant is being classified. This may take a few minutes."`
- On complete: `"Your dashboard data has been updated."`
- On failed: `"Review the output below for error details."`

**TerminalOutput** (reused, same component):
- Height: `h-96` (384px — taller than default h-80 to accommodate longer classify output)
- `onStop` aborts the SSE stream only (no disconnect call)
- Stop button label: `"Stop"` (existing TerminalOutput behaviour)

**Post-completion in-card actions** (rendered below terminal after `status !== 'running'`):
- On `completed`: `<Button variant="default" onClick={() => navigate('/')}>Go to Dashboard</Button>` — full width, `mt-4` (fallback if toast dismissed)
- On `failed`: `<Button variant="outline" onClick={handleCancel}>Start Over</Button>` — full width, `mt-4`
- On `stopped`: `<Button variant="outline" onClick={handleCancel}>Start Over</Button>`

**Toast** fires immediately when status transitions to `completed` or `failed`:
- Handled via `sonner` — `toast.success(...)` / `toast.error(...)`
- See Toast spec below.

### Sidebar: Connection Status Indicator

**Placement:** `<footer>` element added at the bottom of `Sidebar.tsx`, after the `<nav>` block. Uses `mt-auto` to push it to the bottom.

**Expanded sidebar (220px):**
```
<footer className="border-t border-border px-3 py-3 min-h-[44px] flex items-center gap-2">
  <span className={cn(
    "w-2 h-2 rounded-full shrink-0",
    connected ? "bg-[oklch(0.68_0.14_145)]" : "bg-muted-foreground/40"
  )} />
  <span className="text-xs text-muted-foreground truncate">
    {connected ? tenantName : "Not connected"}
  </span>
</footer>
```

**Collapsed sidebar (56px):**
```
<footer className="border-t border-border py-3 flex items-center justify-center min-h-[44px]">
  <span
    title={connected ? `Connected: ${tenantName}` : "Not connected"}
    className={cn(
      "w-2 h-2 rounded-full",
      connected ? "bg-[oklch(0.68_0.14_145)]" : "bg-muted-foreground/40"
    )}
  />
</footer>
```

**Data:** `GET /api/connect/status` → `{ connected: boolean, tenantName: string | null }`
**Polling:** Fetch on sidebar mount. Refresh after wizard completion (via `ConnectPage` emitting a custom event or via TanStack Query invalidation). No continuous polling.

**Dot colours (explicit, not via token name):**
- Connected: `bg-[oklch(0.68_0.14_145)]` (same value as `--tier-user` in globals.css — no new token needed)
- Not connected: `bg-muted-foreground/40`

### Toast Notifications

**Library:** `sonner` (add via `npx shadcn add sonner`) — official shadcn component.
**Position:** Bottom-right (sonner default; do not change)
**Duration:** 8 seconds (longer than default 4s to allow user to click action)
**Placement:** `<Toaster />` component in `App.tsx` root layout.

**Success toast (classification complete):**
```ts
toast.success("Classification complete", {
  description: "Your dashboard data has been updated.",
  duration: 8000,
  action: {
    label: "Go to Dashboard",
    onClick: () => navigate("/"),
  },
})
```

**Error toast (classification failed):**
```ts
toast.error("Classification failed", {
  description: "Classification completed with errors — check the output above.",
  duration: 8000,
})
```

**No action button on error toast** — user stays on the wizard page to read terminal output.

## Accessibility Notes

- All interactive elements have `aria-label` or text content — no icon-only buttons without labels
- Step indicator circles: `aria-label="Step {n}: {label} ({state})"` on each circle element
- Cancel button: visible text label (no icon-only)
- RBAC list: each row is a `<label>` wrapping the checkbox — full row is clickable
- Toast: `sonner` provides accessible `role="status"` announcements natively
- Sidebar status dot (collapsed): `title` attribute provides tooltip context
- Minimum click/tap target for sidebar footer: `min-h-[44px]`
