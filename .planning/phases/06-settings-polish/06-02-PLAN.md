  <name>Task 1: Fix terminal line spacing and \r stripping</name>
  <files>gui/client/src/components/commands/TerminalOutput.tsx, gui/client/src/pages/ConnectPage.tsx, gui/client/src/pages/RunCommandsPage.tsx</files>
  <read_first>
    - gui/client/src/components/commands/TerminalOutput.tsx (full file — find `leading-relaxed` on line 73)
    - gui/client/src/pages/ConnectPage.tsx (find all `toHtml(event.data)` calls — lines 216, 272)
    - gui/client/src/pages/RunCommandsPage.tsx (find `toHtml(event.data)` call — line 151)
  </read_first>
  <action>
    **1. Fix `TerminalOutput.tsx` line spacing:**

    In the `<pre>` element's className (line 73), change `leading-relaxed` to `leading-normal`:

    Before: `className="h-80 overflow-y-auto p-3 text-xs font-mono text-green-300 leading-relaxed whitespace-pre-wrap break-all"`
    After:  `className="h-80 overflow-y-auto p-3 text-xs font-mono text-green-300 leading-normal whitespace-pre-wrap break-all"`

    This changes line-height from 1.625 (`leading-relaxed`) to 1.5 (`leading-normal`), which is appropriate for dense terminal output.

    **2. Strip `\r` in `RunCommandsPage.tsx`:**

    At line 151, change:
    ```typescript
    const html = converterRef.current.toHtml(event.data);
    ```
    to:
    ```typescript
    const html = converterRef.current.toHtml(event.data.replace(/\r/g, ''));
    ```

    **3. Strip `\r` in `ConnectPage.tsx`:**

    At line 216 (auth stream), change:
    ```typescript
    const html = authConverterRef.current.toHtml(event.data);
    ```
    to:
    ```typescript
    const html = authConverterRef.current.toHtml(event.data.replace(/\r/g, ''));
    ```

    At line 272 (classify stream), change:
    ```typescript
    const html = classifyConverterRef.current.toHtml(event.data);
    ```
    to:
    ```typescript
    const html = classifyConverterRef.current.toHtml(event.data.replace(/\r/g, ''));
    ```

    **Root cause:** PowerShell on Windows emits `\r\n` line endings. SSE delivers them as-is. `ansi-to-html`'s `toHtml()` does not strip `\r`, and `whitespace-pre-wrap` renders the carriage return as a visible gap. The `.replace(/\r/g, '')` strips all carriage returns before HTML conversion.
  </action>
  <acceptance_criteria>
    - `gui/client/src/components/commands/TerminalOutput.tsx` contains `leading-normal` and does NOT contain `leading-relaxed`
    - `gui/client/src/pages/RunCommandsPage.tsx` contains `.replace(/\\r/g, '')` on the `toHtml` line
    - `gui/client/src/pages/ConnectPage.tsx` contains `.replace(/\\r/g, '')` on BOTH `toHtml` lines (auth and classify streams)
    - `cd gui/client && npm run build` succeeds (TypeScript compiles)
  </acceptance_criteria>
  <verify>
