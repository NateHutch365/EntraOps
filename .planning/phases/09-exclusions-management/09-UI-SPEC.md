## Interaction States

### Remove Flow

1. User clicks `Remove` button on a table row
2. `<Dialog>` opens with confirmation copy (see Copywriting Contract above)
3. Dialog footer: `Cancel` (variant="outline") + `Remove` (variant="destructive") buttons — 8px gap
4. On confirm: `DELETE /api/exclusions/:guid` → optimistic row removal + toast on success/error
5. On cancel: Dialog closes, no change

### Loading State

- Render 5 `<Skeleton>` rows inside `<TableBody>` while `isLoading` is true
- Skeleton row: `h-10 rounded-md` cells matching column widths

### Empty State

- Render inside `<TableBody>` as a single `<TableRow>` with `<TableCell colSpan={4}>`
- Centered vertically: `text-center text-muted-foreground py-12`
- Two lines: heading (14px/500) + body (14px/400 mt-1)

### Error State

- Render above table, matching existing pattern: `<p className="text-sm text-destructive px-6 py-4">`
- Pattern: `Failed to load exclusions: [error message]`

## API Shape (backend contract)

These endpoints must be implemented to support the page:

### `GET /api/exclusions`
Response:
```json
[
  {
    "guid": "4a3f1b2c-...",
    "displayName": "John Smith",
    "tier": "ControlPlane"
  }
]
```
- `displayName`: resolved from `PrivilegedEAM/**/*.json` GUID field scan; `null` if unresolvable
- `tier`: resolved tier string or `null`
- Source file: `Global.json` → `[0].ExcludedPrincipalId[]`

### `DELETE /api/exclusions/:guid`
- Reads Global.json, removes GUID from `ExcludedPrincipalId` array, writes atomically (temp → rename)
- Response `204 No Content` on success
- Response `404` if GUID not found
- Response `500` with `{ error: string }` on write failure
