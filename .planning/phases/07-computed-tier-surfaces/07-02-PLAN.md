**2. Replace the tier column cell renderer (the `ObjectAdminTierLevelName` column definition, around line 63):**

Change from `cell: ({ getValue }) =>` to `cell: ({ row }) =>` to access `row.original.Classification`. The full updated column definition:

```typescript
{
  accessorKey: 'ObjectAdminTierLevelName',
  header: 'Tier',
  cell: ({ row }) => {
    const tier = row.original.ObjectAdminTierLevelName;
    if (tier === 'Unclassified') {
      const computed = computedTierName(row.original);
      if (computed) {
        return (
          <Badge
            variant="outline"
            className={cn('text-xs font-medium border-dashed', TIER_BADGE_CLASS[computed])}
          >
            {computed}
          </Badge>
        );
      }
    }
    return (
      <Badge
        variant="outline"
        className={cn('text-xs font-medium', TIER_BADGE_CLASS[tier] ?? '')}
      >
        {tier}
      </Badge>
    );
  },
},
```

Key implementation details per user decisions:
- `tier === 'Unclassified'` check first — applied-tier objects short-circuit to the existing solid badge (OBJ-03: unchanged behavior)
- `computedTierName(row.original)` returns the lowest non-Unclassified tier or null
- If computed tier exists → dashed-border badge colored by that tier's color token (OBJ-01, OBJ-02)
- If computed tier is null (empty Classification[] or all Unclassified) → falls through to solid grey "Unclassified" badge (unchanged)
- `border-dashed` is a single Tailwind utility — no custom CSS needed
- `TIER_BADGE_CLASS[computed]` reuses the same color tokens as solid badges (same visual colors, different border style)
- MUST use `row` not `getValue` — Pitfall 2 from RESEARCH: getValue() only returns the accessor value string, not the full row
